API למערכות ימות המשיח

כל הפקודות מהעמוד המרוכזות במקום אחד, עם הסבר פשוט, פרמטרים, תגובות שרת ודוגמאות שניתן להעתיק.

מה זה API?

דרך שבה אתר, שרת או מערכת אוטומציה שולחים פקודה למערכת הטלפונית ומקבלים תשובה.

איפה מגדירים?

הפעולה נשלחת בדרך כלל מהשרת שלך. קבצים, שלוחות, רשימות ומספרים עצמם נמצאים במערכת הטלפונית.

איך בנויה כתובת?

לכל פקודת API יש כתובת משלה. הכתובת מתחילה ב־https://www.call2all.co.il/ym/api/, ובסופה מוסיפים את שם הפקודה. לדוגמה: https://www.call2all.co.il/ym/api/RunTzintuk.

אבטחה: אין לפרסם טוקן אמיתי בדף HTML ציבורי. באתר אמיתי שולחים את הבקשות מהשרת ושומרים את הטוקן במשתנה סביבה.

מה זה משתנה סביבה?

מקום סודי להגדרות של השרת

משתנה סביבה הוא ערך ששמור בהגדרות השרת, מחוץ לקובצי האתר הציבוריים. הקוד יודע לקרוא אותו, אבל מי שפותח את האתר בדפדפן לא רואה אותו.

במקרה שלנו: במקום לכתוב את הטוקן האמיתי בתוך הקוד, נותנים לו שם, למשל YEMOT_TOKEN, ושומרים את הערך האמיתי בלוח הניהול של השרת.

  1. נכנסים ללוח הניהול של שירות האחסון.
  2. מחפשים אזור בשם Environment Variables, Variables או Secrets.
  3. יוצרים משתנה בשם YEMOT_TOKEN ומדביקים בו את הטוקן האמיתי.
  4. בקוד קוראים את המשתנה בלי לכתוב את הטוקן עצמו.
דוגמה ב־Node.js: const token = process.env.YEMOT_TOKEN;

חשוב: GitHub Pages מציג HTML בלבד ואין בו שרת ששומר סודות. לכן אסור להכניס טוקן אמיתי למדריך או לקוד JavaScript שרץ בדפדפן.

הסבר על זיהוי רנדומלי:
השירות מיועד למשל כדי לבצע אימות מספר טלפון באתרים וכד, בעת ציון של ערך זה לא ניתן לשלוח לרשימה של מספרים והמספר הראשון ברשימה כפי שנבחרה יקבל את השיחה.
בתגובה מהשרת יתמלא גם ערך verifyCode שמכיל את 4 הספרות האחרונות של הזיהוי ממנו יצא השיחה, ובערך callerId בתגובה מהשרת יהיה כתוב RAND. תוכלו לבצע אימות של הלקוח בצורה שאתם מבקשים ממנו להקיש את ה4 ספרות האחרונות של המספר ממנו הוא קיבל את השיחה ואתם משווים את זה עם verifyCode.
במקרה שהשיחה לא מזיהוי רנדומלי ערך verifyCode יהיה ריק.
