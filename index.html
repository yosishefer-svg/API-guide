API של ימות המשיח

הפעלת צינתוק בצורה פשוטה וברורה

מדריך מעשי לפיצ'ר RunTzintuk: מה מגדירים במערכת הטלפונית, מה מגדירים בשרת, ואיך נראית קריאת API בפועל.

מה הפיצ'ר הזה עושה?

הפקודה RunTzintuk מפעילה צינתוק למספר טלפון אחד, לכמה מספרים, לרשימת תפוצה, או לרשימת צינתוקים חינמיים.

צינתוק הוא שיחה קצרה שיוצאת למנוי ומתנתקת אחרי מספר שניות. אפשר להשתמש בזה להתראות, אישורי פעולה, תזכורות, או אימות מספר טלפון.

השורה החשובה: את הצינתוק לא "מגדירים בשלוחה" כמו תפריט רגיל. מפעילים אותו על ידי קריאת API לשרת של ימות המשיח.

איפה מגדירים את זה?

במערכת הטלפונית / ניהול ימות

מה נמצא בצד של ימות?

  • הטוקן או פרטי ההתחברות שמאפשרים גישה ל-API.
  • המספר הראשי של המערכת והמספרים המשניים.
  • רשימת צינתוקים חינמיים, אם רוצים לשלוח עם tzl:LIST.
  • רשימת תפוצה, אם רוצים לשלוח עם tpl:LIST.
  • זיהוי יוצא מאושר, אם לא משתמשים בברירת המחדל.
בשרת / באתר / בקוד שלך

מה נמצא בצד שלך?

  • הקוד ששולח בקשת HTTP אל RunTzintuk.
  • המספרים או מספר הרשימה שאליהם רוצים לשלוח.
  • בחירת זמן הצינתוק, אם רוצים לשנות מברירת המחדל.
  • שמירת הטוקן בצורה מאובטחת, למשל בקובץ הגדרות או משתנה סביבה.
לא מומלץ לשים טוקן בתוך דף HTML פתוח ללקוח. אם מדובר באתר, הקריאה צריכה להישלח מהשרת שלך ולא ישירות מהדפדפן של המשתמש, כדי שהטוקן לא יהיה חשוף.

המיקום המדויק של ההגדרות

מה רוצים לעשות? איפה מגדירים? מה שמים בקריאת ה-API?
לשלוח צינתוק למספרים רגילים בשרת שלך בלבד. אין צורך להגדיר רשימה במערכת. phones=0500000000:0520000000
לשלוח לרשימת תפוצה קיימת את רשימת התפוצה יוצרים ומנהלים במערכת ימות. phones=tpl:123
לשלוח לרשימת צינתוקים חינמיים את רשימת הצינתוקים החינמיים יוצרים ומנהלים במערכת ימות. phones=tzl:100
לשלוח לכמה רשימות צינתוקים חינמיים כל הרשימות צריכות להיות קיימות במערכת ימות. phones=tzl:100,200,300
להשתמש בזיהוי יוצא מיוחד הזיהוי צריך להיות מאושר או קיים במערכת ימות. callerId=0771234567
אימות מספר בעזרת זיהוי רנדומלי לא מגדירים מספר קבוע. משתמשים בערך מיוחד. callerId=RAND

סדר העבודה המומלץ

  1. מכינים טוקן API.

    הטוקן הוא האישור שמאפשר לשרת שלך לבצע פעולות במערכת.

  2. מחליטים למי שולחים.

    מספרים ישירים, רשימת תפוצה tpl, או רשימת צינתוקים חינמיים tzl.

  3. אם משתמשים ברשימה, יוצרים אותה במערכת ימות.

    אחר כך לוקחים את מספר הרשימה ומשתמשים בו בפרמטר phones.

  4. שולחים קריאת API מהשרת שלך.

    השרת פונה לכתובת https://www.call2all.co.il/ym/api/RunTzintuk.

  5. בודקים את התגובה.

    אם התקבל responseStatus=OK, הבקשה התקבלה. אם יש errors, בודקים איזה מספר נכשל ולמה.

מה המספר שיופיע בצינתוק?

כאן יש בלבול נפוץ, אז חשוב לעשות סדר פשוט:

למי מתקשרים?

phones

זה המספר שמקבל את הצינתוק. אם כתבת כאן 0500000000, אז הצינתוק יישלח למספר הזה. 

phones=0500000000
מה יופיע על הצג?

callerId

זה המספר שמופיע אצל האדם שמקבל את הצינתוק. כלומר, זה לא היעד של הצינתוק, אלא הזיהוי שממנו הצינתוק נראה כאילו יצא. 

callerId=0771234567
בשפה הכי פשוטה: phones = למי לצנתק.
callerId = איזה מספר יופיע לו על הצג.

אם לא מגדירים callerId

אם לא שולחים בכלל את הפרמטר callerId, המערכת תשתמש בדרך כלל במספר הראשי של המערכת כזיהוי יוצא.

אם רוצים שיופיע מספר מסוים

מוסיפים לקריאה את הפרמטר callerId עם המספר שרוצים שיופיע: 

callerId=0771234567

המספר הזה צריך להיות מספר שמותר למערכת להוציא ממנו שיחות, למשל המספר הראשי, מספר משנה, או מספר שאושר במערכת.

אם רוצים מספר אקראי לצורך אימות

משתמשים בערך המיוחד RAND: 

callerId=RAND

במקרה כזה יופיע למקבל הצינתוק מספר אקראי, וה-API יחזיר לשרת את 4 הספרות האחרונות שלו בתוך verifyCode. את 4 הספרות האלה אפשר לבקש מהמשתמש להקיש.

המספר האקראי לא נבחר מתוך רשימה שאתה מכניס בפרמטר phones, וגם לא מתוך רשימה רגילה שאתה שולח בקריאת ה-API. זהו זיהוי אקראי שהמערכת של ימות בוחרת בשבילך לצורך האימות.

אם אתה רוצה לשלוט בדיוק איזה מספרים יופיעו על הצג, אל תשתמש ב-RAND. במקום זה שלח callerId רגיל, או שלח JSON שבו לכל מספר יעד מוגדר callerId משלו.

חשוב לדעת: את המספר שמופיע בצינתוק לא שמים בתוך phones. את המספר שמופיע בצינתוק שמים ב-callerId.

הפרמטרים החשובים

פרמטר חובה? הסבר פשוט
token כן הטוקן של המערכת שלך.
phones כן למי שולחים את הצינתוק: מספרים, רשימת תפוצה, או רשימת צינתוקים חינמיים.
callerId לא איזה מספר יופיע על הצג של מי שמקבל את הצינתוק. אם לא שולחים, המערכת משתמשת בזיהוי הראשי.
TzintukTimeOut לא כמה שניות הצינתוק יצלצל. ברירת מחדל 9, מקסימום 16.

דוגמאות ברורות

דוגמה 1: צינתוק למספר אחד

בדוגמה הזאת 0500000000 הוא המספר שמקבל את הצינתוק. לא הגדרנו callerId, ולכן יופיע בדרך כלל המספר הראשי של המערכת. 

https://www.call2all.co.il/ym/api/RunTzintuk?token=TOKEN&phones=0500000000

דוגמה 2: צינתוק לכמה מספרים

כאן כל המספרים שבתוך phones יקבלו צינתוק. 

https://www.call2all.co.il/ym/api/RunTzintuk?token=TOKEN&phones=0500000000:0520000000:0530000000

דוגמה 3: צינתוק עם זמן צלצול של 12 שניות

https://www.call2all.co.il/ym/api/RunTzintuk?token=TOKEN&phones=0500000000&TzintukTimeOut=12

דוגמה 4: צינתוק לרשימת צינתוקים חינמיים

את הרשימה עצמה יוצרים במערכת ימות. אחרי שיש לך את מספר הרשימה, למשל 100, שולחים כך: 

https://www.call2all.co.il/ym/api/RunTzintuk?token=TOKEN&phones=tzl:100

דוגמה 5: צינתוק לכמה רשימות צינתוקים חינמיים

https://www.call2all.co.il/ym/api/RunTzintuk?token=TOKEN&phones=tzl:100,200,300

לפי התיעוד, אפשר לשלוח עד 50 רשימות בקריאה אחת.

דוגמה 6: צינתוק לרשימת תפוצה

אם קיימת במערכת רשימת תפוצה שמספרה 123: 

https://www.call2all.co.il/ym/api/RunTzintuk?token=TOKEN&phones=tpl:123

דוגמה 7: צינתוק עם זיהוי יוצא מסוים

בדוגמה הזאת 0500000000 מקבל את הצינתוק, ועל הצג שלו יופיע 0771234567. 

https://www.call2all.co.il/ym/api/RunTzintuk?token=TOKEN&callerId=0771234567&phones=0500000000

דוגמה 8: אימות מספר בעזרת זיהוי רנדומלי

בדוגמה הזאת 0500000000 מקבל צינתוק ממספר אקראי. 

https://www.call2all.co.il/ym/api/RunTzintuk?token=TOKEN&callerId=RAND&phones=0500000000

במקרה כזה התגובה תחזיר verifyCode. זהו קוד של 4 ספרות אחרונות של המספר שממנו יצאה השיחה. אפשר לבקש מהלקוח להקליד את 4 הספרות ולאמת מול הערך שהשרת החזיר.

דוגמה לקוד בשרת

זו דוגמה בסיסית ב-JavaScript לשרת Node.js. בפועל עדיף לשמור את הטוקן במשתנה סביבה ולא לכתוב אותו גלוי בתוך הקוד. 

const token = process.env.YEMOT_TOKEN;
const phone = "0500000000";

const url = new URL("https://www.call2all.co.il/ym/api/RunTzintuk");
url.searchParams.set("token", token);
url.searchParams.set("phones", phone);
url.searchParams.set("TzintukTimeOut", "9");

const response = await fetch(url);
const result = await response.json();

if (result.responseStatus === "OK") {
  console.log("הצינתוק נשלח", result);
} else {
  console.log("שגיאה בשליחת הצינתוק", result);
}

איך נראית תגובה מהשרת?

שדה משמעות
responseStatus אם הפעולה הצליחה, בדרך כלל יופיע OK.
verifyCode מתמלא רק כאשר משתמשים ב-callerId=RAND.
callerId הזיהוי שממנו יצאה השיחה.
callsCount כמה שיחות צינתוק יצאו בפועל.
bilingPerCall חיוב לכל שיחה.
biling סך החיוב.
errors מספרים שנכשלו והסיבה, למשל חסום, כפול, או מספר לא תקין.

סיכום קצר

אם אתה שולח למספרים רגילים: אין כמעט מה להגדיר במערכת. פשוט שולחים קריאת API מהשרת עם phones.

אם אתה שולח לרשימת צינתוקים חינמיים: הרשימה עצמה צריכה להיות קיימת במערכת ימות. בקריאת ה-API משתמשים ב-tzl:מספר_רשימה.

אם זה חלק מאתר או מערכת אוטומציה: הקריאה צריכה לצאת מהשרת שלך, לא מדף HTML פתוח, כדי לא לחשוף את הטוקן.

חשוב: החלף תמיד את TOKEN בטוקן האמיתי שלך, אבל אל תפרסם אותו בדפדפן או בקוד צד-לקוח.
